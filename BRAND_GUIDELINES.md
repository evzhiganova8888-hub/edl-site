## Зачем этот документ

Один источник правды по визуальной идентичности. Если что-то на сайте, в презентации, в PDF, в социальных сетях расходится с этим документом — расходится оно, а не документ.

Этот документ дополняет, но не заменяет: `EDL · Источник Правды v2.0` (позиционирование и tone of voice), `EDL · Strategic Brief v1.2` (стратегия сайта) и `EDL · Design System v1.0` (детальные UI-токены и компоненты).

## 1. Логотип — система из двух марок

В EDL используется **два логотипа** для разных контекстов. Это решение зафиксировано 26 апреля 2026.

### 1.1 Symbol mark (V2) — для digital-продуктов

**Где используется:**
- Сайт: header, footer, favicon
- Telegram-бот: аватар
- Любые цифровые продукты EDL
- Email-подпись (опционально)

**Как выглядит:** круг с симметричной дугой внутри. Чистая, абстрактная геометрия. Хорошо читается на размерах от 16px до 200px.

**Конструкция:**
```
viewBox: 0 0 32 32
Circle: cx=16 cy=16 r=14, stroke 1.5
Path: M 5.5 22 C 5.5 8, 26.5 8, 26.5 22 (stroke 2, round caps, no fill)
```

**Файл-источник:** `logo.svg` в корне репо сайта.

### 1.2 Wordmark (V6) — для документов и презентаций

**Где используется:**
- PDF-документы (коммерческие предложения, отчёты, брифы)
- Презентации (Keynote, PowerPoint, Google Slides)
- Визитки и печатная продукция
- Заглавный лист любых официальных документов
- Watermark на изображениях (опционально)

**Как выглядит:** строгий типографический «EDL» без символа. Только буквы, без дополнительных графических элементов.

**Конструкция:**
- Шрифт: Inter SemiBold (font-weight: 600)
- Letter-spacing: -0.04em (tight)
- В печатных и презентационных контекстах допустим Inter Bold (700) при необходимости большего веса

**Файл-источник:** `logo-wordmark.svg` в корне репо сайта.

### 1.3 Когда что использовать — простое правило

| Контекст | Логотип |
|---|---|
| Цифровой продукт (сайт, бот, приложение) | **V2 — Symbol mark** |
| PDF, печать, презентация, визитка | **V6 — Wordmark** |
| Соцсети — аватар | V2 (квадратное обрамление) |
| Соцсети — обложка, OG-image | V6 (или V2 + V6 в композиции) |

Если сомневаетесь: цифровой → V2, печатный → V6.

### 1.4 Что НЕ делать с логотипом

- ❌ Не растягивать диспропорционально (только пропорциональное масштабирование)
- ❌ Не добавлять тени, обводки, градиенты, эффекты свечения
- ❌ Не менять цвет произвольно (см. раздел 1.5 — допустимые цвета)
- ❌ Не вращать
- ❌ Не использовать на занятом фоне (фотография без фильтра, узорчатая поверхность)
- ❌ Не комбинировать V2 и V6 случайно — только в утверждённых композициях

### 1.5 Цвета логотипа

| Контекст | Цвет |
|---|---|
| На светлом фоне | `#15151D` (почти-чёрный, основной текст) или `#1A1B2E` (брендовый navy) |
| На тёмном фоне | `#F5F5F0` (off-white) |
| В монохромной печати | Чёрный 100% или белый 100% — ничего между |
| На accent-цвете (lime) | `#15151D` (почти-чёрный) — для контраста |

Никогда: лаймовый, цветной, полупрозрачный, dotted, outlined-only-stroke (стирает идентичность V2).

### 1.6 Минимальные размеры

- **V2 (symbol):** минимум 16×16 пикселей. Ниже — теряется читаемость дуги.
- **V6 (wordmark):** минимум 60 пикселей по ширине. Высота буквы E — минимум 14px.

### 1.7 Охранное поле (clear space)

Свободное пространство вокруг логотипа должно быть не меньше:
- **V2:** высота окружности (то есть круг + столько же места со всех сторон)
- **V6:** высота заглавной «E» (с каждой стороны)

В этом пространстве не размещать другие графические элементы и текст.

## 2. Цветовая палитра

### 2.1 Основная палитра

| Роль | HEX | Использование |
|---|---|---|
| **Brand Navy** | `#1A1B2E` | Header, footer, тёмные секции, бренд-марка |
| **Brand Navy Deep** | `#11121F` | Footer base, deep contrast |
| **Brand Navy Soft** | `#2A2B40` | Hover states, тонкие границы на тёмном |
| **Background Primary** | `#FAF9F6` | Off-white. Основной фон контента |
| **Background Elevated** | `#FFFFFF` | Карточки, модалы |
| **Background Tinted** | `#F2F0EA` | Секции для лёгкой смены ритма |
| **Text Primary** | `#15151D` | Почти-чёрный. Заголовки и body |
| **Text Secondary** | `#4A4A52` | Подзаголовки, second-level |
| **Text Tertiary** | `#888893` | Caption, мета, мелкие подписи |
| **Text on Dark** | `#F5F5F0` | Текст на тёмных секциях |
| **Accent Lime** | `#C8FF00` | **Только** CTA, ключевые цифры, активные состояния |

### 2.2 Правила использования accent

Electric lime — высокоэнергетичный цвет. Используется **дисциплинированно**:

✅ **Можно:**
- Кнопки CTA (фон)
- Подчёркивание ключевых слов в hero (под текстом)
- Цифры цены в выделенных карточках
- Active states у элементов
- Тонкая accent-полоса (3px) в OG-image и брендовых документах

❌ **Нельзя:**
- Большие площади заливки (целая секция)
- Цвет тела body-текста
- Градиенты от lime к чему-либо (читается как «крипто»)
- Полупрозрачные оверлеи lime поверх изображений
- Тени в lime

### 2.3 Запрещённые цвета

В EDL нет места: розовому, бирюзовому, оранжевому, коричневому, фиолетовому, жёлтому. Палитра — навигационная серьёзность + один заряженный акцент. Дополнительные цвета размывают идентичность.

Исключение: статусные цвета в UI (success-зелёный, warning-янтарный, error-красный) — только в системных сообщениях, не в маркетинговом контенте.

## 3. Типографика

### 3.1 Шрифты

| Роль | Шрифт | Обоснование |
|---|---|---|
| **Display (заголовки)** | Inter | Open-source, отлажен для tech-продуктов, отличная типометрика |
| **Body (основной текст)** | Inter | Тот же шрифт. Унифицирует ритм страницы |
| **Mono (цифры, технические элементы)** | JetBrains Mono | Парный к Inter, tabular numerals |

В дальнейшем — переход на **Geist** (от Vercel) для display. Geist даёт более характерные геометрические формы, лучше работает с tech-инфраструктурным позиционированием. Это улучшение запланировано на v1.5, не блокер для v1.

### 3.2 Размерная шкала

См. раздел 3.2 в `EDL_Design_System_v1.md` — там полная fluid-шкала через `clamp()`.

### 3.3 Веса

**Только два веса:** Regular 400 для body, SemiBold 600 для заголовков и акцентов.

Не использовать: Light (300), Medium (500), Bold (700), ExtraBold (800). Они нарушают визуальную дисциплину.

Исключение: Mono для цифр может быть Medium (500) для лёгкого подчёркивания.

### 3.4 Запрещённые приёмы

- ❌ ALL CAPS на длинных заголовках (только на eyebrow-метках типа «01 · ПОДХОД»)
- ❌ Italic для целых блоков (italic — только смысловое выделение внутри текста)
- ❌ Подчёркивание текста как декорация (только лаймовая полоса под слоганом в hero)
- ❌ Letter-spacing > 0.1em на body (нечитаемо)

## 4. Изображения и фотографии

См. полные правила в `EDL_Design_System_v1.md` раздел 6. Краткая суть:

### 4.1 Что используем

- **Реальные скриншоты** системы EDL (CRM, бот, утренняя сводка). С обезличенными именами клиентов, реальными цифрами.
- **Реальные фото** Екатерины и команды (на v1 — без фото, добавляются в v1.5+ при наличии профессиональной съёмки).
- **Чистая типографика** — там, где не нужны изображения, работают слова + воздух.

### 4.2 Что не используем

- ❌ Stock-фото (Unsplash, Shutterstock, Pexels)
- ❌ AI-сгенерированные изображения (Midjourney, DALL-E, Stable Diffusion)
- ❌ Абстрактные 3D-объекты, blob-формы
- ❌ Mockups устройств (MacBook, iPhone) с лежащим скриншотом — сразу читается как stock
- ❌ Плейсхолдер-изображения. **Лучше пустой блок с типографикой, чем placeholder**

### 4.3 Принцип

Если нет реальной фотографии или скриншота — не вставляем placeholder. Заменяем визуальный блок на типографический. Это профессиональнее, чем pretend-визуал.

## 5. Tone of Voice

См. полностью в `EDL · Источник Правды v2.0`, раздел 6.

Краткие принципы для брендинга:

- **Деловой, но человечный.** Не корпоративные сухари. Не «привет, друг!». По-взрослому.
- **Конкретика > Абстракция.** Цифры, сроки, состав команды, конкретные артефакты.
- **Активный залог.** «Соберём за 4 недели», не «будет собрано в течение четырёх недель».
- **Имя «Екатерина».** Не «Элис», не «Катя» в публичных материалах.
- **Точечная постирония.** 2–3 раза на странице, не больше. Нигде в названиях продуктов или ценах — только в «человеческих» местах.

## 6. Применения — типовые композиции

### 6.1 Favicon (16, 32, 180, 192, 512px)

Использует **V2 в брендовом navy на квадратном фоне**. Конструкция в `favicon.svg`. Подробности конвертации — см. `MAINTENANCE_GUIDE.md`.

### 6.2 OG-image (1200×630)

См. `og-image.svg`. Композиция:
- Фон: brand-navy (#1A1B2E)
- Top-left: V6 wordmark (40px, off-white)
- Под wordmark: lime accent line (3px, 100px длиной)
- Center-left: 3-строчный headline, последнее слово выделено lime
- Bottom-left: тег «ОПЕРАЦИОННАЯ СИСТЕМА БИЗНЕСА» (caption-размер, серый)
- Bottom-right: URL elephantdreams.ru (caption-размер, серый)

### 6.3 Презентация (Keynote / PowerPoint / Slides)

- **Title slide:** brand-navy фон, V6 wordmark по центру, под ним название презентации body-размером
- **Content slides:** off-white фон, V6 в углу (top-left, 32px), черный текст
- **Section divider slides:** brand-navy фон, типографика как hero-блок сайта
- **Closing slide:** brand-navy фон, V6 + URL elephantdreams.ru + контакты

### 6.4 PDF-документ (КП, бриф, отчёт)

- **Cover:** brand-navy фон, V6 + название документа + дата
- **Tilte page** (после cover): off-white фон, table of contents
- **Body pages:** off-white фон, V6 в footer-маленький, нумерация страниц справа
- **Closing:** контакты + повтор V6

### 6.5 Визитка

Сторона лица:
- brand-navy фон
- V6 wordmark по центру вертикально
- Под ним: имя сотрудника + роль (Inter, off-white)

Оборотная сторона:
- off-white фон
- Контакты (telegram, email, calendly) типографически на Inter
- elephantdreams.ru тонким текстом снизу
